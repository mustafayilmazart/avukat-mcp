# Security Policy

## Reporting a Vulnerability / Güvenlik Açığı Bildirimi

**English:**
If you discover a security vulnerability, please **do not** open a public issue. Instead, email **medya@kesif.org** with:
- Description of the vulnerability
- Steps to reproduce
- Affected version(s)
- Suggested fix (if any)

We will acknowledge within **72 hours** and aim to release a fix within **14 days** for critical issues.

**Türkçe:**
Bir güvenlik açığı bulduysanız, **lütfen herkese açık bir issue açmayın**. Bunun yerine **medya@kesif.org** adresine şunları içeren bir e-posta gönderin:
- Açığın tanımı
- Tekrar üretme adımları
- Etkilenen sürüm(ler)
- Varsa önerilen düzeltme

Kritik bulgulara **72 saat içinde** yanıt verilir, **14 gün içinde** düzeltme yayımlanır.

## Supported Versions

Yalnızca **en son major sürüm** güvenlik düzeltmeleri için desteklenir.

## Out-of-Scope / Kapsam Dışı

- Üçüncü taraf bağımlılıkların açıkları (doğrudan upstream'e bildirin)
- Kullanıcının kendi `.env` veya `config.json` dosyasını yanlış yapılandırması
- Hukuki tavsiye disclaimer'ının yorumlanmasıyla ilgili sorular (legal advice değildir)
